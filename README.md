Cara Menggunakan Project Ini untuk Simulasi
Jangan hanya menyuruh siswa mengetik kode. Gunakan pendekatan "Roleplay":

1. Fase Daily Standup (10 Menit Awal)

Minta siswa membuka file HTML ini.

Anda sebagai Product Owner memberikan perintah:

"Tim, hari ini klien minta fitur Login dan Register. Silakan pecah jadi tugas-tugas kecil dan masukkan ke kolom To Do."

Siswa mengetik: "Buat HTML Login", "Buat CSS Login", "Buat Database".

2. Fase Sprint (Coding Time)

Siswa klik tombol "Maju" untuk memindahkan kartu ke In Progress.

Artinya: "Saya sedang mengerjakan ini sekarang."

3. Fase Definition of Done (Akhir Sesi)

Siswa hanya boleh memindahkan kartu ke kolom Done jika kodenya benar-benar jalan tanpa error.

Jika masih error tapi dipindah ke Done, Anda sebagai "QA Tester" berhak memarahi (bercanda) mereka: "Ini masih bug kok sudah Done? Balikin ke In Progress!"


Apa yang Sebenarnya Mereka Pelajari? (The Hidden Curriculum)
DOM Traversal: Mereka belajar konsep parentElement. Ini logika penting saat nanti belajar React/Vue atau manipulasi data kompleks.

State Logic: Memahami if (kolomSekarang === 'col-todo'). Ini dasar algoritma perpindahan status.

Satisfaction Loop: Efek visual kartu berpindah memberikan dopamin instan yang membuat mereka merasa produktif.
